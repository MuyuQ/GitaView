# Product

## Register

product

## Users

GitaView is for developers who maintain many local Git repositories across
Windows and macOS. They use it as a lightweight desktop utility while working,
and need to understand repository health without opening each project or
interrupting their current task.

## Product Purpose

GitaView provides an at-a-glance view of managed repository status, then lets
users inspect exact Git relationships and take a small set of safe repository
actions. Success means a user can quickly identify repositories that need
attention, open the relevant location, and run guarded fetch, pull, or push
operations with confidence.

## Brand Personality

Quiet, precise, trustworthy. The interface should feel like a well-made desktop
instrument: compact enough to stay present, clear enough to scan quickly, and
calm enough to use throughout the day.

## Anti-references

GitaView should not resemble a marketing dashboard, a neon terminal theme, or a
decorative card gallery. Avoid oversized typography, gratuitous glass effects,
heavy shadows, novelty controls, and visual treatments that compete with
repository status.

## Design Principles

- Put repository health first. Status, counts, and actionable differences must
  be the easiest information to scan.
- Earn attention. Use emphasis only for current selection, risk, errors, and
  actions that matter.
- Preserve context. Expanding the widget, filtering, and opening actions should
  keep the user oriented.
- Make safety visible. Potentially modifying Git operations must be clearly
  labeled and explicitly confirmed.
- Respect desktop flow. Keep controls compact, predictable, and fast for
  repeated use.

## Accessibility & Inclusion

Target readable light-mode contrast and complete keyboard focus treatment.
Never communicate status by color alone; pair colors with text labels, counts,
and screen-reader labels where compact presentation requires them. Hover and
focus states must not shift layout. Respect reduced-motion preferences, support
Chinese-first readability, and keep controls usable for users who cannot rely
on fine pointer movement.
